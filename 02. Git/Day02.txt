=> GIT RESET
git reset moves the HEAD pointer backward
But how much data it keeps or deletes depends on
	-- soft reset
	-- hard reset

GIT SOFT RESET
What it does is
	moves HEAD to previous commit
	keeps changes in staging area
	keeps files unchanged

git reset --soft <CommitID>

GIT HARD RESET
What it does is
	moves HEAD backwards
	clears staging area
	deletes working directory changes

git reset --hard <CommitID>

One-line;
Soft reset = undo commit, keeps the work
Hard reset = undo commit, deletes the work

=> GIT REVERT
it will undo a commit safely by creating a new commit
git revert doesn't delete any commit, it will create a new commit + keeps full commit history

Feature						git reset				git revert
changes history				yes					no
creates a new commit			no					yes
